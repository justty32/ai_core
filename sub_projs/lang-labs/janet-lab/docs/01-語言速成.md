# 01 · 語言速成

Janet 是 Lisp 方言：一切都是 `(函式 參數...)`，用括號分組。以下每段都在 1.41.2 上跑過。

## 括號家族 `()` `[]` `{}` 與 `@`（Janet 跟傳統 Lisp 最不一樣的地方）

傳統 Lisp 只有 `()`；Janet 額外把 `[]` 和 `{}` 當成**內建語法**，這也是它「處理 JSON 很順」的
根本原因（見 [03-json.md](03-json.md)）。心法：**`()` 是「呼叫」，`[]`/`{}` 是「資料」。**

```janet
(+ 1 2)              # () → 當函式呼叫 => 3
[1 2 3]              # [] → tuple（不可變陣列），會求值內容 => (1 2 3)
[(+ 1 1) (* 2 3)]    # [] 裡的元素照樣求值 => (2 6)
{:a 1 :b 2}          # {} → struct（不可變表）=> {:a 1 :b 2}
{:a (+ 40 2)}        # {} 的值也求值 => {:a 42}
```

前綴 `@` = 可變版：

| 寫法 | 型別 | 可變 |
|------|------|------|
| `[1 2 3]` | tuple | ✗ |
| `@[1 2 3]` | array | ✓ |
| `{:a 1}` | struct | ✗ |
| `@{:a 1}` | table | ✓ |

所以 `{:a 1 :b [1 2 3]}` 這種「長得像 JSON 的東西」在 Janet 裡是合法的字面資料，可以直接寫、
內含計算值。細節與各型別操作見 [02-資料結構.md](02-資料結構.md)。

## 綁定：def / var / let

```janet
(def x 10)          # 不可變綁定
(var y 20)          # 可變綁定
(set y 30)          # 改 var
(let [a 1 b 2]      # 區域綁定，只在 body 內有效
  (+ a b))          # => 3
```

`def`/`var` 是全域或檔案層級；`let` 是區域。慣例：優先 `def`，真的要改才 `var`。

## 函式

```janet
(defn add [a b] (+ a b))       # (add 3 4) => 7
(defn greet [&opt who]          # &opt 之後是可選參數
  (default who "world")         # who 沒給就填預設
  (string "Hello, " who "!"))
(defn sum [& xs] (reduce + 0 xs))  # & 收集剩餘參數成 tuple

(def double (fn [n] (* n 2)))   # 匿名函式 fn
(map |(* $ 2) @[1 2 3])         # |(...) 是 short-fn，$ 是第一個參數 => @[2 4 6]
```

## 條件

```janet
(if (> x 5) "big" "small")      # if：一真一假
(when (> x 5) (print "big"))    # when：只有真、body 可多句
(cond                            # cond：多分支
  (< x 0) "neg"
  (= x 0) "zero"
  "pos")                         # 最後一個沒條件 = else
(case x 10 "ten" 20 "twenty" "other")  # case：值比對
```

真假值：只有 `nil` 和 `false` 是假，其它（含 `0`、`""`、空陣列）都是真。

## 迴圈與序列

`loop` 做副作用、`seq` 做推導（回傳陣列）：

```janet
(loop [i :range [0 3]] (prin i " "))   # 印 0 1 2
(each v @[:a :b] (prin v " "))          # 走訪值，印 a b
(seq [i :range [0 5] :when (odd? i)]    # 推導 + 過濾
  (* i 10))                             # => @[10 30]
```

`:range [0 5]` 是 0..4；`:when` 過濾；`:in coll` 走訪集合的值；`:pairs t` 走 key-value。

## 常用內建

```janet
(map |(* $ $) @[1 2 3 4])   # => @[1 4 9 16]
(filter even? @[1 2 3 4])   # => @[2 4]
(reduce + 0 @[1 2 3 4])     # => 10
(length @[1 2 3])           # => 3
```

## 印東西：print vs pp

**重要地雷**：`print` 對陣列 / table 只會印出位址 `<array 0x...>`，不是內容。要看內容用
`pp`（pretty-print）或格式動詞 `%q`：

```janet
(print @[1 2 3])            # <array 0x...>  ← 不是你要的
(pp @[1 2 3])               # @[1 2 3]        ← 這才對
(printf "%q" @[1 2 3])      # @[1 2 3]        ← 內嵌到字串時用 %q（可讀表示）
(printf "%p" @[1 2 3])      # @[1 2 3]        ← %p 亦可（pretty）
```

`%q`／`%p`／`%j`／`%n` 都是印**Janet 表示法**（`%j` 是單行、不是 JSON——別搞混）。
要印**真 JSON** 只有一條路：`(print (json/encode x))`，見 [03-json.md](03-json.md)。

```janet
# 井號到行尾是註解
(defn f

  [x] x)
```
