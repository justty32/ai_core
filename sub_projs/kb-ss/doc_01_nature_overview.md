# doc_01 — 本質定位概觀

> **一句話講完**：這套系統為什麼要從 Terminal (命令列) 開始？因為那是程式最純粹、最沒有廢話的地方。

---

## 為什麼從 Terminal 出發？

「執行一個函式」這件事，不管是在 Python 裡面呼叫、在命令列執行、還是在雲端跑，本質都是一樣的：**給輸入，拿輸出**。

我們選擇從 Terminal (POSIX) 開始，是因為：
1. **介面統一**：輸入就是 stdin，輸出就是 stdout，出錯就是 stderr。
2. **沒框架干擾**：不用管什麼 Web 框架或 UI 庫，直接對著邏輯硬碰硬。
3. **限制明確**：在命令列能跑通的邏輯，以後要擴展到哪裡都容易。

**硬道理**：
- 我們只針對 **POSIX 系統** (Linux, macOS)。
- **不考慮 Windows**。我們要在 Pipeline (水管) 的世界裡玩到極致。

---

## 本質到底是什麼？ (施工中)

這套東西目前可以看作是一個「函式執行組織框架」。我們還在討論它最終會變成一個工具箱、一種設計模式、還是一個完整的作業平台。

但現在，請把它當成：**一套讓 LLM 能夠安全、穩定地調用各種功能的基礎設施**。
