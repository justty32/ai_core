# code_02 — 原型工具集 (`try_implement/tools/`) 說明書

> **一句話講完**：這是一堆還在「實驗階段」的小工具（Indexer, Router, SFC 等），用來測試 `ai_core` 的各種點子。它們都遵循「九軸」規範，而且**通通沒有外部相依**（只用 Python 內建庫）。

---

## 1. 大家的共同規則

1. **一定要有 `--metadata`**：所有工具都得支持這個指令，印出自己的規格。沒了這個，整個系統就串不起來。
2. **Import 是安全的**：你可以隨便 `import` 這些工具，它們不會在 Import 時亂動你的系統（除非你真的去執行它們）。

---

## 2. 核心工具大集合

### `indexer.py` — 掃描器
- **幹嘛用的**：去資料夾裡把所有的程式（`.py`, `.sh`）抓出來，一個個問它們 `--metadata` 是什麼，最後整合成一張「技能清單」（JSON 或 Markdown）。

### `router.py` — 轉接頭
- **幹嘛用的**：一個簡單的對照表。你跟它說「我要跑 upper」，它去查 `routes.json` 發現對應的是 `funcs/upper.py`，就幫你執行它。

### `switch.py` — 智慧轉接頭
- **幹嘛用的**：比 Router 聰明一點，會看條件。例如：如果是 C 語言就跑 C-Linter，如果是 Python 就跑 Py-Linter。

### `sfc.py` — 小函式中心 (Small Function Center)
- **幹嘛用的**：這是一個「萬能工具箱」。為了不讓檔案多到爆炸，我們把一堆幾行的小程式（Tiny Functions）通通塞進去。
- **特色**：
    - 支持 Python 代碼或 Shell 腳本。
    - **Forge 模式**：可以變成一個長駐的 Server，用 JSON 跟它溝通。
    - **動態增刪**：可以在執行時加進新的小函式。

### `hub.py` — 技能中心
- **幹嘛用的**：把 Indexer 掃出來的東西轉成「LLM 看得懂的說明」。
- **重點**：它會根據 LLM 的「預算（Token 數）」自動縮減說明，避免 LLM 被長文撐死。

### `llm_entry_manager.py` — LLM 門房
- **幹嘛用的**：所有對 LLM 的請求都要經過它。
- **為什麼要它**：因為 LLM 很貴又很慢（Singleton 資源），它負責排隊、限流（Rate Limit），確保不會把錢噴光。
- **長駐模式**：可以用 `--socket` 啟動，讓所有小工具共用同一個限流器。

### `idea.py` — 靈感捕捉器 (實戰 Dogfood)
- **幹嘛用的**：這是我們第一個「真正在用」的工具。
- **功能**：幫你把口述的爛點子「洗乾淨（clean）」、整理成「筆記（notes）」、找出「漏洞（critique）」或「發散想法（expand）」。

### `chain.py` — 串接器
- **幹嘛用的**：把一堆 Function 像水管一樣接起來（A 丟給 B，B 丟給 C）。它還能幫你算出這整條「水管」組合起來後的 Metadata 是什麼。

---

## 3. 發現的坑（待解決）

在寫這些原型時，我們發現了幾個問題：
- **語意不明**：現有的「九軸」雖然很完整，但 LLM 讀起來還是有點吃力，可能需要多一條「描述軸」。
- **代碼重複**：Router, Switch, Chain 裡面有一部分程式碼（找執行路徑那段）長得很像，以後可能要抽出來。
- **安全性**：SFC 執行 Python 代碼目前只是「薄弱防護」，還不是真正的安全沙箱。
