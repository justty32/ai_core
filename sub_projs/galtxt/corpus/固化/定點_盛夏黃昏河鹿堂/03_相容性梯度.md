# 相容性梯度 —— 軸與軸怎麼搭（A × B × C × D）

← [README](README.md)｜A 軸細目：[02_軸表_話題入口](02_軸表_話題入口.md)

> [00 §四](00_定點採樣.md) 觀察 5 的落實：座標軸之間不是自由組合，有**相容性梯度**。
> 記號：**✓**＝自然（首選）／**△**＝可，但要多一拍鋪墊／**✕**＝怪（預設禁；但見 §四「好的違反」）。

## 一、A × B（話題 × 誰跨半步）

| | B1 她讓渡 | B2 他遞物 | B3 無遞物 |
|---|---|---|---|
| A1 推理文庫 | ✓ 推薦＝讓書 | △ 他書識不如她，讓反了¹ | ✓ |
| A2 車票梅花章 | ✕ 那是她的寶，不會讓² | ✕ 他無物可遞 | ✓ 安靜的話題 |
| A3 飲料 | ✕ | ✓ 遞無糖紅茶＝整格的存在理由 | △ 只聊不遞＝空轉 |
| A4 貓 | ✕ 無物可讓 | ✕ | ✓ |
| A5 後庫 | ✓ **讓渡的是特權**³ | ✕ | △ |
| A6 怪談 | △ 讓的是「推理」（情報） | ✕ | ✓ |

> ¹ 反向讓書可成立的特例：他讓的不是「書識」而是「機會」（同一本兩人同時看上→他放手）——那其實是 C 軸反差戲，要走 △ 鋪墊。
> ² A2×B1 的「✕」有個階 4+ 的例外：她把收集的舊車票**送**一張——那是劇本級的重大讓渡，不屬日常池。
> ³ **B 軸的「物」泛化成「讓渡物」，正式定為四類**（[陽溜亭驗證 H5](../定點_陽溜亭_初雪傍晚/01_驗證報告.md) 補齊第四類）：**實物**（書、飲料、可樂餅）／**特權**（准入：後庫；優待：三顆白玉、「算你便宜」——特權在半主場的形態是優待）／**情報**（推理、書評、「被記住」）／**勞務・時間**（留下考她單字——無物可遞時讓渡的是自己的時間）。**前提條款：超出服務基線才算讓渡**——基線內的付出（免費續茶誰來都有）不載關係訊號，超出基線的差分才算（基線的機械判準仍 open，見 [90 乙-4](90_問題與討論.md)）。泛化後 B 軸從「遞東西」升級成「誰為誰付出了什麼」，通用性高得多。

## 二、A × C（話題 × 反差格）

| | C1 隨便＋挪懷 | C2 沒什麼特別＋摩挲 | C3 你先走＋目光 | C4 要來就來＋在等 |
|---|---|---|---|---|
| A1 文庫 | △ | ✓ 推薦後的遮掩 | △ | ✓ 接進新書日 |
| A2 車票 | ✓ 寶物被看見的遮掩 | ✓ | △ | ✕ 安靜話題配元氣鉤＝斷裂 |
| A3 飲料 | ✕ 飲料不會「挪懷」 | △ | ✓ 被在意到之後的逃 | △ |
| A4 貓 | ✕ | △ | △ | ✓ 「你和貓不一樣」的餘波 |
| A5 後庫 | △ | ✓ 特權遮掩「只是爺爺順便」 | ✕ | ✓ |
| A6 怪談 | ✕ | ✓ 口頭禪主場 | ✕ | △ |

**規律浮現**：C2（口頭禪遮掩）幾乎全行——它是秋穗的**萬用反差格**；C1/C3 綁定「有實物／有離場」的話題。反差格不是四選一均勻分布，是**一個主格＋三個條件格**。

## 三、A × D（話題 × 收尾）

| | T8 散場 | T9 定格 | T10 鉤子 |
|---|---|---|---|
| A1 文庫 | ✓ | △ | ✓ 讀完再來 |
| A2 車票 | △ | ✓ 幾十年並排的兩枚章＝定格畫面自帶 | ✕ |
| A3 飲料 | ✓ 出去喝＝天然散場 | △ | △ |
| A4 貓 | ✓ | △ | ✓ 進新書日 |
| A5 後庫 | △ | △ | ✓ 下次帶你進去…嗎 |
| A6 怪談 | △ | ✓ 謎留著 | ✓ 下次驗證推理 |

**規律**：話題自帶「溫度」——安靜話題（A2/A6）天然落 T9，社交話題（A1/A4/A5）天然落 T10。**收尾不是獨立選擇，是話題溫度的函數**：D ≈ f(A)——[陽溜亭驗證 H6](../定點_陽溜亭_初雪傍晚/01_驗證報告.md) **支持**，D 軸從自由軸降級為派生值、殘餘空間再砍一軸。**修正項**：半主場存在外生收尾觸發（營業尖峰、收店）——完整式為 **D ≈ f(A) ⊕ 場所鬧鐘**；河鹿堂全場由人物意志收尾，是「**無鬧鐘場所**」的特例（鬧鐘欄見上層 [04](../04_槽填充模板.md)【地點】表）。

## 四、「好的違反」問題（本檔最重要的懸念）

✕ 格預設是錯。但戲劇史上最亮的瞬間常是**刻意違反**：A2×T10（她對著幾十年前的車票，忽然元氣地說「下次我們去坐霞川線吧」）——若寫得出來，那是**角色突破自己**的重大時刻，比任何 ✓ 格都動人。

```
定案（2026-07-17，使用者定調「純程式終點」後）：✕ 格不是禁格，是「高價格」——
  嚴格度不做開關，做【代價模型】：
    硬約束（護欄/canon/越權/稱謂階）永遠擋
    相容性 ✓＝0 分、△＝小額、✕＝高額，數值化進成本函數
  量產模式 ＝ 敘事能量預算 0（✕ 買不起，等價於硬擋）
  劇本模式 ＝ 給一筆預算；「好的違反」＝付得起代價的違反
             付費方式可檢查：✕ 格前後有無鋪墊 beat（有＝打折）
```

「固化表擋掉的是錯還是未被鋪墊的對」——答案：**兩者都擋，但後者可以買回來**。詳見 [../README「終點形態」](../README.md)。
