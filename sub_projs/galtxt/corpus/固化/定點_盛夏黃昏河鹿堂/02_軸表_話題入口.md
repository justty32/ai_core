# 軸表：話題入口（A 軸）—— 話題池是算出來的，不是想出來的

← [README](README.md)｜起點：[00_定點採樣](00_定點採樣.md)

> [00 §四](00_定點採樣.md) 觀察 3 的落實：話題入口候選＝**地點鉤子 ∩ 角色鉤子 ∩ 季節允許集**，機械交集、不靠發想。
> 本檔把程序真的跑一遍——結果**比手挑的 A1–A4 多找出兩個入口**（A5 後庫、A6 怪談），證明程序比直覺強。
> 還冒出一個新固化維度：**話題有「階門檻」**。

## 一、原料（兩張鉤子清單＋季節過濾）

**店誌側**（[河鹿堂店誌](../../世界/日和町地誌/河鹿堂店誌.md)）：門鈴／進新書日木牌／文庫本牆／二手打折竹籃／梅花章（前主人痕跡：日期、夾頁物）／櫃台算盤印台／**後庫（私藏書庫）**／二樓相撲轉播／店外三隻流浪貓／七不思議第六則（後庫無名梅花章書）。

**dossier 側**（[柏木秋穗](../../人物/柏木秋穗.md)）：推理口味（昆恩／橫溝／有栖川）／愛昭和文庫黃褐紙墨味／收集舊車票與壓花書籤／**後庫特權**／壞彈簧沙發位／遠觀三隻貓／討厭甜膩罐裝（偏無糖紅茶）／討厭雨天鞋襪／口頭禪與小動作／新生代表・榮譽榜心結／入學考的秘密。

**季節允許集（盛夏）**：蟬鳴、冰飲、悶熱→店內涼、名古屋場所相撲、（夏夜）河鹿蛙。雨具類（梅雨）、圍巾暖桌類（冬）被過濾掉。

## 三、負例（交集外／被門檻鎖住的）

| 話題 | 為何開不起來 | 類型 |
|------|--------------|------|
| 社團 | 她帰宅部 × 店內無社團鉤子——交集為空 | **空集話題**：硬開＝尬聊，生成器直接禁 |
| 天氣寒暄 | 交集非空但空泛（「好熱」誰都能說） | **不推進話題**：可當 B2 開口的墊句，但 B3 不能只靠它 |
| 新生代表／榮譽榜 | dossier 有，但那是她的**心結** | **門檻話題**（3+）：階不夠就碰＝她語域瞬間退回客氣體（可當「踩雷」負樣本素材） |
| 入學考的秘密 | dossier 核心秘密 | **門檻話題**（4+，且需強觸發）：日常語料永遠碰不到，是劇本級素材 |

## 四、階門檻——本檔冒出的新固化維度

話題不只有「開得起來／開不起來」，還有**最低關係階**：

```
話題(topic) := { 交集來源, 階門檻 n, 下游相容集 }
校驗：本場稱謂階 < 話題門檻 → 打回（或改寫成「踩雷」戲——但要旁白標明後果）
```

這和 [02 敬語與稱謂階](../02_敬語與稱謂階.md) 是同一個狀態機的兩面：**稱呼是階的「輸出訊號」，話題可及範圍是階的「權限表」**。合起來，階 n 就從一個數字變成一組可校驗的具體行為。

## 本檔引出的問題（詳見 [90](90_問題與討論.md)）

- 交集程序要不要寫成真程式（輸入兩份 .md、輸出話題池）——第一個可執行固化件的候選（丁-1）
- 階門檻的數字誰定？現在是我拍的——從 34 篇劇本反向統計「什麼話題出現在什麼階」可驗證（丙-2）
- 「不推進話題」（天氣寒暄）在文法裡的地位：垃圾還是潤滑？（甲-2）
