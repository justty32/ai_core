# 定點_盛夏黃昏河鹿堂 —— 單場景細固化實驗台

← [固化/](../README.md)

> 把**一個場景釘死**（盛夏・黃昏・河鹿堂・柏木秋穗・階 2），在這個定點上**把固化推到最細**——beat 文法、話題軸、相容性、句級語彙、旁白預算，一層層往下鑿。
> 目的不只是把這一場寫好：**每鑿一層都會逼出通用機制與新問題**——機制候補搬回上層 [固化/](../README.md) 或畢業去 llm_forge，問題收進 [90_問題與討論](90_問題與討論.md)。
> **這是嘗試性質的實驗台**：允許挖錯方向、允許半途廢案；廢了就進 `archive/`。

## 為什麼用「一個定點」做實驗

上層 [固化/01～06](../README.md) 是**橫向**的通用刀胚（跨場景、跨角色）。但固化能推到多細、推過頭會不會殺死活氣——這種問題在通用層答不了，只能**縱向**在一個具體定點上鑿到底看看。定點選這裡是因為鉤子密度最高：秋穗 dossier × 河鹿堂店誌的交集最厚。

## 產出去向

- **機制被確認通用** → 抽掉場景細節、搬回上層 `固化/`（如 beat 文法、話題交集程序）。
- **機制被確認可執行** → 照 galtxt 鐵律 3，畢業搬 [llm_forge/](../../../../llm_forge/AGENTS.md) 固化成框架件。
- **問題被決斷** → 從 [90](90_問題與討論.md) 移除，決斷落到對應檔案裡。
