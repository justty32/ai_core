# 固化 —— 台詞規則素材庫

> 「固化階梯」的第一批**規則素材**。
> 把 galgame 台詞裡**確定性的規律**，從純 LLM 自由生成，一級級往下踹成確定性的**規則／詞表／模板**。
> 這裡放的是「刀胚」：給後續 pipeline 當**約束、檢核、部分求值**用的原料，不是最終成品。

## 這是什麼

日和町語料目前多由 agy（LLM）自由生成再人工審。自由生成的問題是：
**同一種確定性規律，每次都要 LLM 重新「想」一遍**——既慢又不穩，還會漂移（OOC、油膩、稱呼亂跳）。

「固化」就是把這些規律抽出來，寫成**不需要 LLM 也能執行**的東西：

- 傲嬌角色的句尾就是那幾種 → 寫成**詞表**，不必每次靠模型抓語氣。
- 「傍晚的河堤，某人正在看水」這種開場 → 寫成**槽模板**，填空即可。
- 這個不油膩世界不會出現的詞 → 寫成**禁語表**，當**確定性護欄**直接攔。

固化後，LLM 的工作從「什麼都生」縮小到「只填模板留白、只在護欄內發揮」——這就是**部分求值（把確定的部分先算掉）**。

## 終點形態（2026-07-17 使用者定調）

固化的最終目標：**整條生成管線弄成純程式，執行期不經（或極少經）LLM**。

- **品質優先、單句延遲不設限**——「為了一句高品質好台詞，願意等待」。一句台詞背後的計算複雜度可以很高：**overgenerate-and-rank**（生成端便宜地炸出大量候選、檢核端按規則砍、按成本排序取優）；甚至整場戲當一個搜尋問題解全域最優。
- **確定性**（2026-07-17 追加定調）：生成器＝**純函數** `G(條件) → 台詞`，同條件必同輸出；**條件不含隨機種子**（無 RNG）。推論：①同座標只有一句＝argmax（成本並列用確定性 tie-break，需全序）；②**變化全部上移到條件向量**——殘餘軸是輸入欄位或由世界狀態推導，不是骰子；③**重複感靠「歷史入條件」解**：`G(條件, 已用句歷史)`，去重＝確定性懲罰項，函數仍純（重跑同存檔逐字一致、二週目自然不同）；④**表帶版本、表也是條件**：換表版＝換輸出合法，同版不同輸出＝bug——調品味＝改表資料＝可 review 的 diff；⑤定點採樣的多樣本從「可能輸出們」改役為**成本函數的校準對**（哪句該贏、理由寫成分數）。換來：可重現／可 diff／可 memoize／golden-test（逐位元比對）。
- **五層架構**：L1 場景規劃（座標搜尋）→ L2 beat 展開（文法）→ L3 句規劃（句型選型＋預算簿記）→ L4 表層實現（槽填充查表）→ L5 檢核排序（護欄＋成本加總）。素材對應見 [定點實驗台](定點_盛夏黃昏河鹿堂/README.md)。
- **真源＝規則（程式），表＝快取**（materialized view）：「預鋪 vs 合成」是假二選一——上游（dossier／01–06）一動，快取重算。手工細表（如定點 04）當合成函數的**黃金標準錨**（雙錨驗證的落點）。
- **嚴格度＝代價模型，不是開關**：硬約束（護欄／canon／視角越權／稱謂階）永遠擋；相容性 ✓△✕ 數值化進成本函數；「好的違反」＝付得起代價的違反（鋪墊 beat＝打折）。量產＝敘事能量預算 0，劇本＝給預算讓搜尋器自己花。
- **LLM 移到編譯期**：執行期零 LLM；LLM 只在**建材期**當句庫擴充器（離線炸候選 → 人審 → 蓋章入庫）——ai_forge「鑄刀」的字面實現：鍛造時出力，刀成之後砍東西不需要它。
- 複雜度花在**選擇**（搜尋空間＋成本函數）、不花在**變換**（字串層層改寫）——前者每步可解釋可調試。

## 怎麼用（固化階梯的位置）

```
純 LLM 自由生成
   │  抽規律
   ▼
【本資料夾：規則素材】← 你在這裡
   │  被 pipeline 消費
   ▼
① 生成前：把模板／詞表塞進 prompt 當骨架與約束
② 生成後：拿禁語表／稱謂階／語域表當「檢核器」過濾與打回重寫
③ 更進一步：模板可直接填槽出句，完全不經 LLM（純確定性）
```

## 使用守則

- 這些是**素材**不是**教條**：模板與詞表是為了「省重複勞動」，不是把每句話鎖死。留白處仍交給 LLM，才有活氣。
- **護欄（03）優先級最高**：任何生成——不論套模板或自由發揮——都必須先過禁語／OOC 表。
- 詞表會**漂**：新角色、新場景長出來時回頭補這些表；把它們當**活文件**維護。
- 一切以**不油膩、含蓄**為最終仲裁：規則與基調衝突時，基調贏。
