# 評測：挖空一句台詞（fill-in-the-blank benchmark）

> 用途：為「用笨模型生成台詞、要能自動評分」準備的 benchmark。
> 做法：從一段合背景的真實場景對白裡，**挖掉一句台詞當正解**；日後餵前後文給模型、讓它生成那一句，再與正解比對／打分。

## 為什麼要做

現階段台詞是 agy「廣撒」生成、人工審過（見上層 corpus 的 18 場景）。要進一步「用更笨、更便宜的模型量產台詞」，就得先有一把尺：

- **能不能自動判斷「這句生成得好不好」**——需要成對的（前後文 → 正解）樣本。
- 樣本要**難度分層**：從「只要有提到某道具/敬語階就算對」的護欄型，到「要真的讀懂上下文情緒」的語意型。
- 樣本要**涵蓋面廣**：四季 × 城鎮/學校各地點 × 不同情緒 × 不同女角類型，避免模型只會寫某一種甜。

## 每題欄位說明

- **題號／標題**：`Qxx｜情境一句話`。
- **context**：場景、季節、地點、在場角色、氛圍。
- **說話者設定摘要**：挖空那句是誰講的、其性格與語域（敬語階／口氣）。
- **前後文**：完整對白，挖空處標成 `【★ 空格 ★】`。正解那句的說話者名保留在行首。
- **正解台詞**：折疊在「正解」小節（評測時先蓋住）。
- **評分要點**：三段——(a) 為何這句好、(b) 語氣為何、(c) 該避免的 OOC。
- **難度**：`護欄型` / `中` / `需語意理解`。護欄型另附「護欄關鍵詞」（生成句須命中才給基礎分）。

## 評分建議（給日後打分程式參考）

分三層合分，權重可調：

1. **護欄命中**（硬條件，0/1）：該題若標了護欄關鍵詞（如「傘」「圍巾」「梅花章」「敬語」），生成句沒命中直接扣到底。
2. **語域相符**（0–2）：敬語階、稱呼（同學／姓／名／綽號）、口氣（傲嬌／元氣／文靜）是否與角色設定一致。
3. **語意契合**（0–3）：是否承接前文、鋪墊後文、情緒對；是否守住基調（含蓄、不油膩）。

**基調紅線（任何題答錯即大扣）**：
- 不油膩：不要直球告白、不要肉麻稱呼、不要過度親暱肢體。
- 含蓄：留白優於說白；用道具/景物/動作帶情緒，勝過直接講心情。
- 世界一致：只用日和町世界聖經內的地點/道具/季節；不得出現手機依賴、都市地標、超現實元素。
- 語域一致：委員長不會突然講粗口；傲嬌不會突然坦率黏人；學姐不會用晚輩口氣。

## 使用流程（人工或程式）

1. 取一題，蓋住「正解」小節，把 context + 前後文餵給待測模型。
2. 收生成句。
3. 依「評分建議」三層打分；護欄型先驗護欄關鍵詞。
4. 對照正解與評分要點覆核，記錄分數與失分原因（OOC 類型）。
