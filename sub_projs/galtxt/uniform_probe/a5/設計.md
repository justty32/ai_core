# a5 — 統一節點 kernel（高層戲劇結構 probe）

> probe 目標：驗證「八股／段／句／填充物／原子事實**不是不同種類，只是不同粒度**」——
> 用**同一種節點**把《最後的櫻花雪》**整篇劇本**的高層戲劇結構搭起來（比場景更粗的粒度），
> 高層節點**引用**下層場景節點，未展開者以**懸空引用＝佔位需求**表示。

跑：`sbcl --script a5.lisp`（自帶 assert，全綠才 exit 0）。

## 核心：唯一資料型「節點」

一切事實不論粒度都是 `(node id …plist…)`，欄位皆選配：

| 欄位 | 意義 |
|------|------|
| `:層` | 粒度標（**自由文字、僅註記**；真正層數由 refs 推導＝自然湧現） |
| `:述` | 一句話描述（骨架印這個） |
| `:引` | refs：子節點 id 清單；指向未定義者＝**懸空＝佔位需求** |
| `:順路` | 收束邊（分支線專用）：指向兩線該合一的節點——分支收束護欄靠它機械成立 |
| `:效` | `(lambda (w)…)` 進入節點時對跨場景世界狀態 w 的回寫（季節時鐘／情緒弧） |
| `:護` | `(lambda (w)…)` 節點內建的邏輯判斷／護欄，回 `(values ok why)` |

**內部節點可帶大量邏輯**（`:效` 節奏推移、`:護` 結構決策／收束護欄），全塞進同一型 → 呼應
使用者定調「內部節點帶大量邏輯判斷」。`node` macro 一行把節點壓成 plist 註冊（`:護`/`:效` 折成 lambda，其餘 quote）。

## 用到的粒度層級（場景以上＋接地示範往下）

`全劇（層4）` → `幕（相遇／分支/收束/主旨/尾聲）` → `場景（線A/線B/河堤・談心）` → `段` → `句（層0 葉）`。

- **層數自然湧現**（承 gen_v1 柱四）：葉／懸空＝0，內部＝子最大＋1。骨架印出 `全劇4 > 收束3 > 談心2 > 句0`——越往上越「敘事」、高層最粗。**同一種節點往上疊、往下疊都成立**。
- `:層` 標只是給人看的註記，**不限四種**；真正的粒度是 refs 拓樸算出來的。

## 懸空引用怎麼表示（＝佔位需求）

高層節點的 `:引` 直接寫下層 id；該 id 若沒有對應 `(node …)` 定義，`懸空?` 即為真。
骨架把它印成 `↳ … 〔懸空・佔位需求〕`，`佔位單` 把全部懸空邊收成待展開工單（本 probe 10 筆，如
`相遇 → 天台・開場`）。這正是 gen_v1「懸空連結＝佔位需求，planner 解析到時觸發按需細化」的機械形態——
高層結構**先搭骨、細節後補**，未補處機械可查、絕不憑空接地。**接地不變式**：`接地?` 沿 refs 下行須達
已定義的葉（本 probe 由 `收束→河堤・談心→段→句` 這條鏈接地到層0）。

## 高層邏輯（三條護欄／時鐘全部機械成立）

- **分支收束護欄**：`線A`/`線B` 各帶 `:順路 收束`；`分支` 節點的 `:護` 斷言兩線的 `:順路` 皆存在、
  皆合一到同一節點——**A/B 必因『回車站順路』合一於 收束**（不靠哪條分支實走，是靜態結構護欄）。
- **季節母題時鐘**：`:效` 沿 相遇→收束→尾聲 推 `將盡→落雨→落盡`；`尾聲` 的 `:護` 斷言季史遞進不倒退。
- **情緒弧**：`:效` 推 `自在（相遇）→落寞（收束）→釋懷（主旨）`；`主旨` 的 `:護` 斷言跨場景情緒史等於此序。

時鐘／情緒弧作為**跨場景狀態 w**（帶歷史的軸），由前序 DFS `過帶` 疊出，護欄事後統一驗。

## 一句話結論

**把整場八股再往上疊一層（全劇結構）用同一種節點——成立。** 高層節點就是「refs 指下層、
自身帶 `:效`/`:護`」的普通節點，層數自然湧現、懸空即佔位、收束護欄靜態可判。
**限制**：本 probe 只把高層搭「骨」（下層多為懸空佔位），尚未驗「細化落點回填時對高層護欄的一致性傳播」
（gen_v1 的 backrefs／canon 鎖）；且分支這裡只驗**結構收束**，未驗「單一實走路徑」下 w 只該疊一條分支的效
（本 probe 為展示結構，兩線 `:效` 都跑了，去處疊成 `(河鹿堂 陽溜亭)`）——真跑一輪需在 `過帶` 加選路。
