# a2・統一粒度節點 kernel — 設計註記

> 把《最後的櫻花雪》場景一（舊校舍天台開場）從最粗（整場八股）到最細（原子事實／填充物）
> **全部用同一種「節點」表示**。跑：`sbcl --script uniform.lisp`（自帶 assert，exit 0）。

## 核心主張

八股／段落／句式／填充物／原子事實 **不是四種東西，是同一種節點的不同粒度**。
本檔只有**一種資料**（`*ns*`：`id → (kids . fn)`）與**一個 kernel macro `n`**：

- **葉節點**（原子／填充）：`kids` 為空，`fn` 直接回值；**可帶邏輯判斷**（例 `安靜詞` 用 `pick`
  依 env 選候選、預設回 canon）。
- **內部節點**（句式／台詞／段／八股）：`fn` 遞迴 `rz` 子節點再縫合；**可帶大量邏輯**
  （護欄、成本、分支都寫在 `fn` 裡，見 `選擇肢`）。

粒度**任意深、不限四種**：本場實測 `depth` 0→4 共 **5 層**同時存在，且同一棵樹裡各子樹深淺不一
（`主角登場` 段深度 1、`邀約引子` 段深度 3），完全由素材需要決定，kernel 不預設層數。

## kernel（一個 macro）

```lisp
(defmacro n (id &rest parts)
  `(setf (gethash ',id *ns*)
         (cons ',(%kids parts)
               (lambda (env) (declare (ignorable env))
                 (concatenate 'string ,@(mapcar #'%part parts))))))
```

`n` 沿用 realizer 的 `tmpl`「資料即 list」風味，把一串 `parts` 折成節點：

| part 形態 | 意義 |
|-----------|------|
| 字串 | 字面文字 |
| keyword（`:br`） | 換行 |
| bare symbol | 子節點引用（編譯成 `(rz 'id env)`，遞迴實現） |
| `(form)` | 邏輯判斷：`env`、`rz` 可見、須回字串；護欄可在此 `assert` 後回 `""` 不佔字 |

`%kids` 掃 parts 收子節點 id（bare symbol ＋ form 內 `'quote` 到的 id），只為「查粒度」而存；
`rz` 實現時不靠它，純遞迴。這讓**動態分支的子節點也能被粒度查詢看見**（見下）。

## 遇到「四分法沒有的東西」：選擇肢 / 分支

四分法（八股→段→句式→原子）是**一棵靜態合成樹**，沒有「同一位置、實現取決於執行期選擇、
且互斥」這種東西。選擇肢就是這種——但**不需要新種類**，它只是一個 fn 帶 `if` 的普通節點：

```lisp
(n 選擇肢 (if (eq (g* env :choice :A) :A) (rz '分支A env) (rz '分支B env)))
```

- 它**引用兩個互斥子節點** `分支A`/`分支B`（藉 form 內的 `'分支A`/`'分支B` 被 `%kids` 收錄，
  故 `(kids-of '選擇肢)` 仍看得到兩條分支——粒度查詢不因「動態」而瞎）。
- 與狀態分析一致：A/B **只改「事・物」段**，人時地共有段逐字相同（assert 驗「所以才想逃跑啊」
  兩線都在；`二手推理小說`／`紅豆湯圓` 各分支獨有）。收束回河堤屬場景二，不在本場。

同理，**共享原子** `河名（霞川）` 被 `開場空鏡` 與 `邀約引子` 兩處引用——節點是「引用」而非
「複製」，整體其實是 **DAG** 而非純樹（assert 以 `subtree` 驗兩段都可達 `河名`）。

## 自帶 assert（三類）

1. **能組出整場**：`(rz '天台場景 …)` 得 20 行台詞；含身分交代、邀約引子、伏筆物。
2. **確定性逐位元**：同 env 重算 `string=`；凍結 `csum` 錨定 byte-level；跨行程 `cmp` 亦一致
   （零 LLM／零 RNG／不依賴 hash 迭代序）。
3. **粒度可查**：`depth`＝5 層；`leafp` 原子=葉、八股=內部；`subtree` 驗共享引用；
   `選擇肢` kids 數=2。

## 與 realizer_l 的異同

- **同**：資料即 list、macro 編譯期折 `concatenate`、字串=字面／symbol=引用的同像性。
- **異**：realizer 的 `tmpl`／`*scene*`／`*fill*` 是**分層三張表**（八股格架、句模板、填充）＋
  一條固定 pipeline；此處**塌成單一 `*ns*` 與單一 `n`**——粒度不再是三個寫死的抽象層，而是
  節點間的引用深度，可任意深、可 DAG、分支即節點。護欄／成本從 realizer 的獨立 `ck`／`icost`
  函數，內化成節點 fn 內的邏輯（同一種節點承載）。
