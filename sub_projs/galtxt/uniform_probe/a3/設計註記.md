# a3 — 統一節點 kernel（設計註記）

獨立探針 a3。回應核心洞見：**八股／段落／句式／填充物／原子事實不是不同種類，只是不同粒度**——都是同一種「節點」，用 Common Lisp 的 list 表示；粒度任意、不限四種。

跑：`sbcl --script a3.lisp`（自帶 assert，全綠才印摘要）。

## 一句話設計

一個節點就是一個 list：`(ID :欄 值 …)`，全部丟進一張表 `*G*`（唯一真源）。
- **原子事實**＝最細粒度＝無 `:引` 的節點，**第 0 層**，`:值` 帶命題，可掛邏輯（`:蘊`／`:斥`／`:鎖`）。
- **中／上粒度節點**（關係、場景狀態、戲劇結構）＝有 `:引` 的節點，**層數自然湧現**：`層 = max(被引用節點層)+1`。
- **句式／填充物**也是同一種節點，只是活在 `:域 :字面`（值＝措辭候選／模板，不是命題）。

寫入這張表的唯一動作是 `節` macro；其餘全是讀取端的謂詞。

## kernel（就這一段）

```lisp
(defvar *G* (make-hash-table :test 'eq))                     ; id → 節點（一張表即整張網）
(defmacro 節 (id &rest kv) `(progn (setf (gethash ',id *G*) ',(cons id kv)) ',id))
(defun g (id) (gethash id *G*))
(defun p (id k &optional d) (getf (cdr (g id)) k d))
(defun 引 (id) (p id :引))
(defun 層 (id) (let ((r (引 id))) (if r (1+ (reduce #'max (mapcar #'層 r))) 0)))
(defun 接地 (id) (let ((n (g id))) (and n (every (lambda (r) (and (g r) (接地 r))) (引 id)))))
(defun 查 (pred) ...穩定序 match...)     ; 確定性：sort by symbol-name
(defun 細 (id) ...LOD 取最細子...) (defun 解 (id) (p (細 id) :值))
(defun 誰引 (id) ...backrefs...)
(defun canon一致 (canon) ...排斥邊不同入...) (defun 蘊含閉合 (canon) ...)
(defun 可改? (id) (not (p id :鎖))) (defun 細化合法? (child) ...父須為區間...)
```

`節` macro 把整個 kv plist 存成 literal data（`',(cons id kv)`），所以 DSL 寫起來就是純資料：符號、字串、keyword、符號 list 都不會被求值。這是「最少程式碼、macro 壓縮」的落點——**寫入語法零樣板，讀取端全是小謂詞**。

| 柱三・LOD 細化 | `:父` ＋ `細/解`：resolve 穿透到最細值；`細化合法?` 守「落在父區間內」 |

## 原子事實與「句式/填充物」真的同構嗎？差在哪

**同構於容器，不同構於語義。** 容器層面完全一樣：都是 `(ID …)` list、都有 id、都可被 `:引`、層數同一套機制湧現、同一個 `*G*`、同一個 `查`。差別在**值所屬的域**——事實在**真值域**（值是命題，帶真假，參與 `canon一致`／`蘊含閉合`／canon 鎖這些一致性謂詞）；句式/填充在**字面域**（值是措辭候選/模板，無真假，它的「邏輯」是選擇與折字串的生成函數，一致性謂詞對它無意義）。所以 `真值?` 這條界線把兩者分開：kernel 對兩者一視同仁地存、引、算層，但邏輯謂詞只掃真值域。一句話——**節點是同一種資料結構，事實與句式是它的兩種「域」**。
