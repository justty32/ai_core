# uniform_probe —— 統一節點表示・五路並行探索＋綜合

← [galtxt INDEX](../INDEX.md)｜演算法對照面：[一台搜尋機.md](../一台搜尋機.md)

> **命題（使用者定調）**：八股／段落／句式／填充物／原子事實**不是不同種類，只是不同粒度**——
> 都是同一種「節點」，用 Lisp list 表示；粒度是任意連續層級（**不限四種**）。葉（填充/事實）可帶邏輯，
> 內部節點可帶大量邏輯。**用最少的程式碼**（macro 壓縮）。
> 五個 agent 各自獨立設計最小 kernel＋產一個粒度切片，再由主 agent [綜合](synth/synth.lisp)。

## 五路（各自獨立、全 `sbcl --script` 綠、自帶 assert）

| 探針 | 切片 | kernel 形態 | 行 | 一句發現 |
|------|------|------------|----|----------|
| [a1](a1/) | 河堤全粒度 | `node=(id 層 realize-λ)`＋註冊表；葉vs內部只差 body 有無 `(r 子)` | 138 | **種≠粒度**：同層可同時有旁白(葉)與台詞槽(內部) |
| [a2](a2/) | 天台開場(含選擇肢) | `n` macro：字串/keyword/符號/`(form)` 折進 concatenate | 138 | **分支不需新種類**＝fn 帶 if 的普通節點；結構是 **DAG 非樹**(共享引用) |

| [a5](a5/) | 全劇高層結構 | plist＋`:護`/`:效` lambda | 155 | **層由 refs 拓樸湧現**、懸空引用＝佔位工單；八股再往上疊一層成立 |

## 五路收斂到的共同核心

四分法（八股/段/句式/填充）＋原子事實，全塌成：

- **一張註冊表**（id → 節點）＋**一個寫入 macro**＋**一個遞迴 realize**。
- **葉 vs 內部＝湧現**：只看有沒有引用子節點，非型別。
- **粒度＝湧現**：`層(id)=1+max(層 子)`、葉=0；**不限四種、可任意深**（a5/a2）。
- **邏輯全收進節點**：填充物選詞、分支、母題導出、護欄——都是節點 body 裡的 form（a4/a1）。

## 五個關鍵發現（各路貢獻，綜合時都收進去）

1. **種 ≠ 粒度**（a1）：同一粒度可有不同「種」的節點；種類是假象，粒度是真軸。
2. **分支＝普通節點**（a2）：選擇肢就是 `(? :choice A B)`；兩分支仍是子節點，DAG 而非樹。
3. **兩個域**（a3）：同一容器裝事實與句式；`真值?` 分域，一致性謂詞（canon/排斥/蘊含）只掃真值域。
4. **填充物帶邏輯 → 界線溶解**（a4）：填充物能讀 ctx/遞迴後，「句式」只是「引用得多的填充物」。
5. **層湧現＋懸空＝佔位**（a5）：層數由 refs 算出；引到未定義 id ＝待展開工單（＝gen_v1「懸空連結」）。

## 綜合：[synth/synth.lisp](synth/synth.lisp)（102 行，六 demo 全綠）

收攏五路最佳點的**最小統一 kernel**（核心 ~30 行）：

- **寫入**：兩個 macro——`node`（字面域，realize 回字串）／`fact`（真值域，帶命題）——共用一張 `*ns*` 表。
- **part 折疊**（a4）：字串＝字面｜符號＝子引用(遞迴 `rz`)｜`(? key a b…)`＝依 `*env*` 選一（**填充選詞與分支是同一運算**）｜`(! form)`＝逃生艙（帶邏輯的填充/母題導出）。
- **讀取**：`層`(湧現粒度)／`懸空?`(佔位)／`真值?`(域界線)／`kids`——全從同一表算。
- 六 demo：①一種節點跑整場 ②粒度湧現 0/1/2/3 ③真值域vs字面域同表分域 ④分支=普通節點 ⑤懸空=佔位 ⑥換 `*env*`＝換風格另一整場。

**比原本省掉四套資料結構＋四套接縫**（realizer_l 的 `*scene*/*tmpl*/*fill*/*變項動詞*` 四表 → 一表一 realize）。

## 表示 × 演算法：合一

- 本夾＝**表示**（統一節點）；[一台搜尋機.md](../一台搜尋機.md)＝**演算法**（gen_v0 引擎 ≅ director 是同一台搜尋機）。
- 接縫：synth 的 `*env*`（母題/情緒/選路）**就是搜尋維度**——`(? key …)` 的每個選擇點都是一個待搜的 env 值。
  director 的 `best-line`＝**搜 `*env*` 使成本最小**。於是整個系統塌成一句：

  > **一棵統一節點樹 ＋ 一個搜 `*env*` 的 search kernel，遞迴跑在每一層粒度上。**

  gen_v0＝搜 act 粒度的 env｜realizer director＝搜句式/填充粒度的 env｜未來 facts director＝搜事實/act 粒度的 env——**同一台機器**。
