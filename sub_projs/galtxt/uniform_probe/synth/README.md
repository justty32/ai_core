# synth —— 五路綜合・最小統一節點 kernel

← [uniform_probe](../README.md)（五路對照＋五發現）｜落地版：[../../uni/](../../uni/README.md)

> 把 [uniform_probe](../README.md) 五個 agent（a1–a5）各自的最小 kernel **綜合成一支**：
> 一種節點跑所有粒度（八股/段/句式/填充/原子事實），粒度由 refs 湧現、不限四種。
> **102 行、核心 ~30、六 demo 全綠**。這是「表示」端的綜合；「演算法」端見 [../../一台搜尋機.md](../../一台搜尋機.md)。

## 跑

```sh
cd uniform_probe/synth && sbcl --script synth.lisp
```

## 收攏了五路的什麼

| 來源 | 收進 synth 的點 |
|------|----------------|
| a1 | 一張註冊表 `*ns*`＋遞迴 `rz`；葉 vs 內部只是「有沒有引用子節點」 |
| a2 | 分支＝ `(? :choice A B)` 普通節點；粒度查詢不因動態而瞎 |
| a3 | `node`（字面域）/`fact`（真值域）共用一表；`真值?` 分域，一致性謂詞只掃真值域 |
| a4 | part 折疊：字串=字面｜符號=子引用｜`(? key …)`=env 選（選詞＝分支同一運算）｜`(! form)`=邏輯 |
| a5 | 層由 refs 湧現（`層=1+max(子層)`）；懸空引用＝佔位工單 |

## 六 demo

① 一種節點跑整場｜② 粒度湧現 0/1/2/3｜③ 真值域 vs 字面域同表分域｜
④ 分支＝普通節點（env 選路）｜⑤ 懸空＝佔位工單｜⑥ 換 `*env*`＝換風格另一整場。
