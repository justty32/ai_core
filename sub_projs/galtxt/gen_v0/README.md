> **第一把刀胚**：`G(條件) → 台詞` 的純函數雛形。範圍鎖死「盛夏黃昏河鹿堂・柏木秋穗」一個定點的 A1 話題線。
> 使用者指定：**Lua、所有條件以 lua table 表述**。執行期**零 LLM、零隨機**。

## 跑

```sh
cd gen_v0 && lua main.lua     # Lua 5.4+；示範 ①～⑥ 全數自帶 assert
```

## 檔案

| 檔 | 是什麼 |
|----|--------|
| [conditions.lua](conditions.lua) | **條件向量**（G 的唯一輸入）：version／world／cast／relation／scene／history 六組，純資料 |
| [tables.lua](tables.lua) | **規則表**（真源的快取形態）：稱謂階、話題池＋階門檻、相容性定價、beat 序列、預算、**句庫**、護欄 |
| [engine.lua](engine.lua) | **G 本體**：L1 座標（驗門檻＋D≈f(A) 派生＋定價）→ L2 beat 展開 → L3 預算簿記 → L4 選候選＋填槽 → L5 護欄 |
| [main.lua](main.lua) | 六個示範：①基準 ②確定性自檢（逐位元比對）③歷史入條件（同座標全場換次優、骨架不變）④變化上移到條件（ending 改 T9）⑤階門檻硬擋 ⑥表版本檢查 |

## 落實了哪些定調

- **確定性**：無 RNG；只用陣列順序遍歷（不走 hash 遍歷）；成本同分用 id 字典序 tie-break（全序）；示範②逐位元自證。
- **變化全上移到條件**：換台詞的唯一途徑是改 conditions（座標、歷史、階、版本）——示範③④⑤⑥各展示一個維度。
- **歷史入條件**：`history.used_lines` 施確定性懲罰（+100），同座標二週目自動落到次優候選，函數仍純。
- **表也是條件**：`version.tables` 不符直接拒絕。
- **代價模型**：相容性 ✓=0／△=2／✕=10，`narrative_budget=0`＝量產模式（✕ 買不起即硬擋）；階門檻與護欄是永遠的硬約束。
- **LLM 移編譯期**：句庫是唯一的「創意物」，執行期只查——目前手工從定點 00 樣本抽的，之後由 LLM 離線炸候選＋人審入庫。

## v0 的已知簡化（＝v1 的工作清單）

1. **逐拍貪心**不是全場最優：預算簿記按 beat 順序 threading；v1 換全場搜尋（單句不計延遲的授權還沒用滿）。
2. 句庫只鋪了 A1/B1/C2/T9+T10 一條線（每拍 2 候選）——夠示範機制，不夠鋪產能；擴句庫是資料工作不是程式工作。
3. beat 文法用展開後的固定序列＋條件拍，還沒實作產生式與 c1–c6 約束檢查。
4. 規則表是手抄快取；丁-4 機制定稿回寫完成後，應改為從 corpus 生成（真源＝規則）。
5. 洩底「恰一」靠 B4B5 候選自帶（leak=1 記帳），還沒做「反差拍必有且僅有」的結構性校驗。
