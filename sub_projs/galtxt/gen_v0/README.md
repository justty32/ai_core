# gen_v0 —— 確定性台詞生成器・玩具版（Lua）

> **第一把刀胚**：`G(條件) → 台詞` 的純函數。範圍鎖死「盛夏黃昏河鹿堂・柏木秋穗」定點，句庫兩條線（A1 讓書・甜／A2 車票・靜）。
> 使用者指定：**Lua、所有條件以 lua table 表述**。執行期**零 LLM、零隨機**。
> 引擎已到 **v1**：整場戲當一個搜尋問題解全域最優。

## 跑

```sh
cd gen_v0 && lua main.lua     # Lua 5.4+；示範 ①～⑧ 全數自帶 assert
```

## 八個示範各釘一條定調

| # | 示範 | 證明什麼 |
|---|------|----------|
| ① | 明給座標 A1/B1/C2、ending 由 D≈f(A) 派生 | 低階 API＝編譯器餵源碼 |
| ② | 同條件重跑逐位元相同 | G 是純函數（無 RNG、陣列遍歷、嚴格 < 擇優） |
| ③ | ①的句子登記進 history 再跑 | 歷史入條件：同座標換次優、骨架不變字面全換 |
| ④ | 座標全空＋`intent="靜"` → 搜尋收斂到 A2/B3/C1/T9（216 座標試遍、5 可行） | 高階 API：整場戲＝搜尋問題；trace 誠實報告覆蓋率 |
| ⑤ | 座標全空＋`intent="甜"` → A1/B1/C2/T10 | 同上，意圖不同解不同 |
| ⑥ | 硬點 A2×T10（✕格定價 10）：預算 0 被擋、預算 12 放行＋trace 標「⚠付費違反」 | 代價模型：好的違反＝付得起代價的違反 |
| ⑦ | 階 1 開 A5 後庫話題 | 階門檻＝永遠的硬約束 |
| ⑧ | 條件要求舊版表 | 表也是條件 |

## v1 相對 v0 的升級

- **全場窮舉取代逐拍貪心**：預算是全場約束，貪心會漏最優；DFS＋下界剪枝，毫秒級（「單句不計延遲」的授權開始用了）。
- **beat 產生式**：`場景 := B0 B1 B2 B3+ (B4) B5 (B6) B7` 依 c2（B6 要有東西可蓋）/c3（B6 可在反差前後）/c4（無讓渡→B5 獨立拍）展開變體，變體結構成本（第二輪獎勵、棄標點拍罰款）一起進總成本。
- **L1 搜尋**：留空的座標軸全枚舉；門檻硬過濾、✕ 格對預算收費、intent 成本引導。
- **requires_pick 承接鏈**：候選可宣告「前面某拍必須選了特定句」（「沒有。」必須接在「書拿反了嗎」後）——句際連貫性的硬約束，搜尋自己找出合法承接鏈。

## 已知簡化（＝下一步工作清單）

1. 句庫僅兩條線、每拍 2–3 候選——夠示範機制，不夠鋪產能；擴句庫是資料工作（LLM 編譯期炸候選＋人審入庫）。
2. 規則表是手抄快取；丁-4 機制定稿回寫完成後，應改為從 corpus 生成（真源＝規則）。
3. intent 目前只有甜/靜兩檔、成本手拍；該由丙-2 統計 pass 校準。
4. c5 殘響拍、c6 招呼拍（半主場）未實作——等第二個地點進句庫時一起。
5. 洩底「恰一」靠候選自帶 leak 記帳，還沒做「反差拍必有且僅有」的結構性校驗。
