  - **踩坑補**（記 [common/gotchas](workflows/common/gotchas.md)）：**匿名 namespace 的同名 struct 讓 glaze COMDAT section 跨 TU 撞名**——三個 `llm_*.cpp` 各自匿名 namespace 定義同名 `ReqBody`/`RespBody`，GCC 匿名 namespace 一律 mangle 成 `_GLOBAL__N_1`、glaze `quoted_key_v` COMDAT section 跨 TU 撞名（大小不同→折疊後給錯 JSON 鍵，不只警告是潛在 bug）。改各 TU 唯一命名 namespace（`ask_impl`/`tool_impl`/`media_impl`/`json_impl`）解決。
  - **open 待辦（下一步）**：http／glaze／ask 接口＋三擴充（工具/多媒體/結構化）就緒、離線 file:// E2E 全綠。剩：① **接真後端**跑一次真 round-trip＋SSE 串流（與 try_1/try_2 共用那關，見 [WAIT_USER](WAIT_USER.md)）；② 由 `llm::Client` 欄位**反射生成 `--flag` CLI**（對照 try_2 從 schema 生 cli.lua；try_3 改從 struct 反射——「不搬 schema」延伸到 CLI 那一哩）。可選深化：三擴充補串流、strict schema 的 `required` 對齊真後端。Linux/Manjaro 需在 `vcpkg.json` 加 `curl` 並改 preset 路徑（README 有記）。
