# digest — 讀長文 / 資料做消化（工作流入口）

← [WORKFLOWS](../../WORKFLOWS.md)｜[INDEX](../../INDEX.md)

讀長文、報告、影片逐字稿、一堆連結，建立可追蹤的**摘要、索引、出處**。目標是自己看懂並能延續、回訪。這是本工作流的**入口**：先讀本檔（含下面的 index），再進各摘要檔。

> 本工作流已從單檔升級成資料夾型（第一份產物落地時觸發）：本 README 兼「入口 + index」，各摘要落成同層 `*.md`。摘要再變多就照 [DEV-GUIDE](../../DEV-GUIDE.md) 再分 `summaries/`、`themes/`。

## 流程

```
Done when: 指定材料讀完、下面 index 已更新、摘要落檔、關鍵出處可回溯
```

1. 在下面 index 建條目，狀態 `⬜/✅/⏸️`。
2. 取得原文，記來源 URL / 標題 / 日期。
3. 自己讀原文或可驗證抽取；**不要只信搜尋摘要或二手轉述**。
4. 產出摘要檔（照下面模板）；跨材料成主題時整理到一起。
5. 更新 index 條目狀態。

## 摘要模板

```md
# {{標題}} 摘要
## 一句話
## 重點
## 對我的用途
## 來源
- {{URL 或位置}}
```

- 原始來源與摘要衝突時，以原始來源為準。

## 何時不用

- 查一個即時事實、不需留存 → 直接查證回答。
- 要動手產出 → 接 [write](../write.md)；要行動 → 接 [plan-a-thing](../plan-a-thing.md)。
