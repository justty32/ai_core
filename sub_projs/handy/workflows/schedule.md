# schedule — 一次性定時請求（行程，心血來潮）

← [INDEX](../INDEX.md)｜[WORKFLOWS](../WORKFLOWS.md)｜執行由 [tick](tick.md) 每次心跳觸發

**做什麼**：記「**心血來潮、通常一次性**」的定時請求（＝**行程**）——「下午五點重啟 XXX」「今晚 8 點提醒我 OO」這種臨時指定時刻、做一次就沒了的事。與 [routines](routines.md) 相對：routines 是**不常變動、固定循環**；schedule 是**臨時、一次性**。

**兩種進入方式（同一個檔、兩個目的）**：
- **A. 登記**（你說「**請幫我登記行程**：下午五點重啟 XXX」時）→ agent 換算成絕對時刻、寫進下面「一次性時刻表」。**只登記、不當場做。**
- **B. 執行**（[tick](tick.md) 每次心跳叫起時）→ 判時間 → 到點的就做 → 做完刪。

## A. 登記（使用者請求時）

1. 聽使用者口述的「**時刻 + 內容**」。
2. **換算成絕對時刻**（含日期，Asia/Taipei）：相對講法（「五點」「今晚 8 點」「明天」）依現在時間換算。
3. 寫進下面「一次性時刻表」一列（時刻 / 內容 / 執行者）。
4. **回一句確認**：登記了什麼、幾點做。**不當場做**（那是 tick 的事）。

## B. 執行（tick 心跳時）

1. **判當前當地時間**（`TZ='Asia/Taipei' date '+%Y-%m-%d %A %H:%M'`）。
2. **掃下面時刻表**：有沒有「時刻 ≤ 現在」且還沒做的項。
3. **到點就做**：
   - 該項是使用者當初**親自排的** → 視為**已授權**，照做。
   - 仍守 [AGENTS.md](../AGENTS.md) 鐵律精神：情況有變、或會有預期外的不可逆影響 → 先確認再動。
   - **重活**不在心跳裡跑（記下 / 另開 session）。
4. **做完刪掉該列**（一次性，別留著下次重跑）。錯過很久才醒到的 → 看情況補做或跳過並告知使用者。
5. 沒到點的留著、略過。
