# routines — 例行事務 / 定期待辦

← [INDEX](../INDEX.md)｜[WORKFLOWS](../WORKFLOWS.md)｜由 [tick](tick.md) 每次心跳執行

**做什麼**：記錄「在某個**時機 / 週期**下該做的事」（＝**常規事務**）——開工時、收工前、每週、每隔幾天……。這些**不常變動、固定循環**（與 [schedule](schedule.md) 的臨時一次性「行程」相對）。目的：① 怕忘記；② 能交給 **agent 代勞**的就標出來。

**兩種進入方式（同一個檔、兩個目的）**：
- **A. 登記 / 維護**（你說「這是常規事務，**幫我加進 routine**」時）→ agent 寫進下面「時機分區」或「間隔登記表」。
- **B. 執行**（[tick](tick.md) 每次心跳叫起時）→ **看現在時間 → 對照下面清單 → 有到期的就做**。

每個項目標三件事：**執行者**（我 / agent）、**內容**、（間隔型才要）**上次執行日期**。

## B. 執行（tick 每次叫起就照這做）

1. **判當前當地時間**（Asia/Taipei）：
   ```bash
   TZ='Asia/Taipei' date '+%Y-%m-%d %A %H:%M'
   ```
2. **對照清單**：現在落在下面哪個「時機分區」、有沒有到期的間隔項（距上次執行超過週期）。
3. **到期就做**：
   - **唯讀事務**（盤點、健康檢查）→ 當場做（守 [AGENTS.md](../AGENTS.md) 鐵律）。
   - **會造成不可逆影響**的（動機器檔案 / repo 工作樹 / push）→ **先問使用者**，未確認不動。
   - **重活**（深度巡檢 / 調查）→ **不在這裡跑**，記下 / 之後另開 session。
   - **間隔型做完** → 更新登記表該列的「上次執行」。
4. 沒到期的就略過。（提醒機制**先不做**：到期就當場做 / 當場說。）

## 時機分區（live 清單，隨時增 / 改 / 刪）

> 下面是**範例分區**，示範一則項目該標哪三件事。換成你自己的時機。

### 開工（morning，約 09:00 前）
- （範例）**開工唯讀盤點**（執行者：**agent**，唯讀可代勞）：`git -C ~/repo/ai_core status --short`、`curl -s -m3 localhost:1234/v1/models`（LM Studio 是否在）——有異常轉對應處理。

### 收工（EOD，約 17:30）
- （待補：問我還有沒有事；沒事就跑收尾——保存 / 確認要不要 push）
