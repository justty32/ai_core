handy 生長記

一組 folder-as-callable 的小工具——拿現成程式用腳本薄殼包裝、按慣例組合。 LLM 只是其中一個工具；後端換到 DeepSeek。這一輪，它從兩支薄包裝長成一套會自我擴展的 agent 工具集。

一句話 handy 這一輪：① 資料夾改成分層工作流骨架；② 四個薄工具（llme · zhtw · wf · mail）都接上 DeepSeek；③ 用 handy 思想復刻 hermes-agent 成 hermy——一個會自寫技能的 agent，再把它拆成每檔 <50 行的 CLI 小檔、移植到 C++/Lua/CL。
住戶地圖

五個住戶，各自一攤

每個都是獨立可跑、可單獨測的小程式；彼此用 CLI 串連。狀態與載體如下。

llme

多 endpoint dispatcher：llme <endpoint> 換名＝換模型，缺 key 時自動從環境注入。

Fennel · 118 行可用
zhtw

繁中翻譯薄包裝：把取樣參數＋人格烤死，只留輸入。有位置參數用參數、否則讀 stdin（管線公民）。

Fennel · 54 行可用
wf

兩層任務派發器：先啟發式判「要不要動手」，只需腦走 llme、要動手走 claude、 非同步走 mail。

Fennel · 181 行可用
mail

inbox 協議：把太複雜、不想當場等的任務寫成信投遞，mail run 之後逐封交給 claude 處理。

Fennel · 156 行可用
hermy

hermes-agent 的 handy 化復刻：DeepSeek 當腦、技能資料夾當手、會用 create_skill 自寫新技能。

Python · 46 行 + lib可用 MVP
地基

三條照抄的慣例

新住戶站在這三條上，不必每次重想。

folder-as-callable

一個資料夾＝一個可呼叫物：固定名入口＋各自的 config／腳本。技能、endpoint、住戶都是這形狀。

stdin 管線公民

無位置參數就讀 stdin，讓每個工具都能進 unix 管線：cat x | zhtw、… | wf。

部分求值

能用固定腳本判斷／輸出的環節就別呼叫 LLM。wf 用關鍵詞先判路由，清楚的任務 0 個 LLM call。

hermy

用 handy 思想復刻 hermes-agent

不抄它三千個檔，抓本質用 handy 的方式落地：一個會從經驗長出技能的 agent，後端 DeepSeek。

hermes 核心
hermy 的 handy 化
Skills（腳本包成工具、能 create_skill 自產）
folder-as-callable：skills/<n>/＝skill.json＋run；create_skill＝AI 自我擴展
對話迴圈（模型→工具→結果→迴圈）
DeepSeek function-calling 迴圈＋shell-out 執行技能
記憶（對話後同步、檔案化）
memory/log.ndjson append-log
LLM provider
DeepSeek（deepseek-chat）

再拆成每檔 <50 行、可單獨 CLI 呼叫的小檔（unix-composition）：

hermy            編排器（46 行）：把 lib/* 用 CLI 串成迴圈
lib/skills-json  掃 skills/ → OpenAI tools JSON   （發現）
lib/ds-chat      stdin{messages,tools} → 一次 DeepSeek → message （腦）
lib/run-skill    exec skills/<n>/run              （手）
lib/mem-append   append memory/log.ndjson         （記憶）

同一套拆解移植到五語言（ports/{cpp,lua,cl,fennel,s7}/），統一用 curl（HTTP）＋jq（JSON）當現成引擎、語言只當 glue。各版都用真 DeepSeek 驗過： 工具迴圈算出 3^4=81、自我擴展當場自寫 rev 技能把 handy 反轉成 ydnah。

Lisp 家族（cl／fennel／s7）把每個 CLI 收成 5–6 行薄殼——邏輯全推進庫（util.lisp 195／util.fnl 203／util.scm 288）。 cl／fennel 靠載入路徑自算根、薄殼是該語言本身；s7 拿不到自身路徑，薄殼改用 shell shim（算根＋跑 cllm 的 llm-s7）——正是 handy「拿現成用薄殼包」的一貫手法。

另一批

cllm-apps 多語言移植

handy 四工具（llme/zhtw/wf/mail）以 shell-out 方式移植到 Python／C++／Common Lisp， 並照 cllm-apps 房規整合 cllm 周邊工具（anthropic-proxy／llm-login）。放在 ~/code/cllm-apps/<lang>-handy/，本機保留、不收版控（依你決定）。

怎麼跑

回家開一個 shell 就能試

# 前置：設 DeepSeek 金鑰（handy 各工具 auto-inject）
export DEEPSEEK_API_KEY=sk-...

# 翻譯 · endpoint 換模型 · 管線
./zhtw Hello world              # → 你好，世界
echo "..." | ./zhtw
llme deepseek 用一句話介紹你自己

# 派任務：wf 自己判要不要動手（-b 腦 / -a claude / -i 投 inbox）
wf 解釋什麼是 mmap              # 判 brain → llme 直接答
wf 幫我把 foo.py 的錯誤處理補上  # 判 agent → claude 動手

# inbox 非同步交接
mail send 幫我重構 utils.py  →  mail list  →  mail run

# hermy：會自寫技能的 agent（cd hermy 後）
./hermy 用 shell 技能算 2 的 10 次方
