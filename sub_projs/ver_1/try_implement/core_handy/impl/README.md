# impl/ — 設施實作層

> ★ **事實基準（2026-06-28 收斂）**：本目錄的 `*.hpp` 是設施面的**現況實作**。
> 設施面的設計討論在 [`../notes/impl_*.md`](../notes/)（已降為歷史記錄）；與本層衝突時**一律以本層為準**。
> 描述面事實基準在 [`../defs/axes.hpp`](../defs/axes.hpp)。

對應 `notes/00_index.md`「兩層區分」的第 2 類內容：**標準實作庫提供的設施**。

這裡放的是「**替程式做掉**某件事」的功能碼——依託九軸標準、建在其上的設施。
不是描述某函數是什麼（那屬於 [`../defs/`](../defs/)），而是程式可選用、把某種職責**託管**出來的實作。

設施：
- 軸 1 **統一 I/O 通道讀寫**（跨傳輸：檔案/資料夾、pipe/FIFO、shared memory、tcp/http），
  讓作者不必為每種通道各寫一套。
- 軸 4 **狀態統一託管**（標準目錄慣例 config/cache/state/data 的現成實作：路徑解析、建目錄、JSON 讀寫）。
  **是軸 1 檔案/資料夾設施的特化**（限定 4 目錄 + 語意約束），可建於其上。

> 注意：有設施面的軸，其**描述面**仍歸 [`../defs/`](../defs/)（如軸 1 的 `Entry` 型別）。
> 純設施、無描述面的只有軸 4（描述面被軸 3 `stateful` 涵蓋）。
> 此層的內容在九軸**標準定義**裡不存在；它是標準實作庫額外提供的價值。
