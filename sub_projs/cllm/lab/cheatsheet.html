1 心智模型

學一次 API，九語言都會用——binding 只是 llm_ask() 的薄皮，形狀一致、差在語法。

ask(prompt[,ep],…)回完整答案字串
on_delta給了＝串流，逐段回呼（回 truthy 中止）
schema要求結構化輸出（回 JSON 字串）
tools+on_tool送定義；收 {name,arguments}（單輪）
media/modalities輸入媒體／要求輸出模態
on_media模型產出媒體 {mime,bytes}

每個資料夾的 play.* ＝同一套六個示範用該語言寫一遍，直接改它就是開發。

2 llm CLI（unix filter）

llm 用一句話介紹你自己           # 位置參數
echo 數到五 | llm --stream        # stdin＋串流
cat report.txt | llm 總結這份     # 合體：指令在前資料在後
git diff | llm 把 - 寫成 commit    # 「-」明指 stdin 插入點
旗標作用
--stream逐段吐 stdout
--system <文字>system role 指示（字面文字，在 user 前插一則）
--image <檔>輸入媒體（可重複，mime 由副檔名推）
--schema <檔>JSON Schema → 結構化輸出
--tool <檔>工具定義（可重複），tool_calls 一行一則 JSON
--modality n[=檔]要求輸出模態如 audio=cfg.json
--media-out <目錄>產出媒體落檔（目錄須先存在）
--endpoint --model --api-key後端／模型／金鑰
--config <檔>扁平 JSON 設定檔

連線／取樣旗標由 Client 欄位反射生成 → 以 llm --help 為準。退出碼：0 成功 1 用法錯 2 請求失敗 130 Ctrl-C。

3 C · C++

C — <cllm/cabi.h>

llm_client_t c={0}; c.endpoint=url;
llm_request_t r={0}; r.prompt="你好";
llm_handlers_t h={0}; h.on_text=on_text;
llm_ask(NULL,&c,&r,&h); // 回 LLM_OK

編：cc play.c $(pkg-config --cflags --libs cllm jansson)

C++ — <cllm/llm.hpp>（便利層）

llm::Client c; c.endpoint=url;
auto r=c.ask("你好");  // Result<string>
if(r) use(*r); else r.error().message;
c.ask("數到五",{.stream=true,.on_delta=cb});
auto ch=llm::ask_as<Character>(c,"給我角色");

編：g++ -std=c++23 … $(pkg-config --cflags --libs cllm)

⚠ 給反射（ask_as<T>／make_tool）的 struct 要放 namespace scope，函式內 local struct 反射不了。

3 Lua · Fennel

Lua — require("llm")

llm.ask("你好", endpoint)
llm.ask{ prompt="數到五", endpoint=ep,
  stream=true, on_delta=function(p)
    io.write(p) return false end }
-- system/schema/tools/on_tool/on_media/media/modalities
-- JSON: require "dkjson"

Fennel — (require :llm)

(llm.ask {:prompt "數到五" :endpoint ep
  :stream true :on_delta (fn [p] (io.write p) false)})
⚠ Fennel table 鍵一律用底線：:on_delta／:on_tool，不是 :on-delta。跑在 lua 5.4。

3 s7 Scheme · Common Lisp

4 接後端 / 登入

lab 預設打離線 fixture。接真模型三條路：

(a) 本機 LM Studio — 最省事

llm 介紹你自己 \
  --endpoint http://localhost:1234/v1/chat/completions \
  --model local-model
⚠ reasoning 模型別設 --max-tokens：思考鏈與答案共用預算，設太小會讓 content 變空。

(b) Anthropic — 走 anthropic-proxy

cmake --build --preset linux-debug --target anthropic-proxy
./build/tools/anthropic-proxy &
cp tools/anthropic-proxy/configs/opus.json \
   ~/.config/llm/config.json   # 填 sk-ant- key
llm 哈囉

(c) OAuth 登入 — llm-login

cmake --build --preset linux-debug --target llm-login
cp tools/llm-login/providers/openrouter.json \
   ~/.config/llm/oauth.json
./build/tools/llm-login login   # headless 加 --no-browser

5 建置 / 安裝

lab 吃 ~/dev/cllm/ 的產物，本身不建。要重建 cllm 本體：

cd ~/repo/ai_core/sub_projs/cllm
cmake --preset linux-debug        # vcpkg 自動裝 glaze
cmake --build --preset linux-debug
bash install-dev.sh               # 冪等裝進 ~/dev

相依極簡 C++ 只用 vcpkg 的 glaze＋系統 libcurl。preset：linux-*／mingw-*。

Windows（MinGW g++16.1）：已實建——cli_smoke 35/35、cpp/python 庫路徑已驗（cpp 六能力綠、python 五綠一崩）。但無 pkg-config，run.sh／install-dev.sh 那條走不了，需手動組 include／link；libcllm.dll 已改 -static 自足。細節見 README「Windows 註記」＋cllm gotchas/windows。
