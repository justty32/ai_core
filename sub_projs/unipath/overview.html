unipath

「先歸一於路徑、後成局」願景的階段一落地——採 Plan 9 思想 ＋ 9P 協議，以 FUSE 起步，把一個正在跑的 process 的執行態環境暴露成一棵可 walk 的路徑樹。

方案 A（Plan 9＋9P 蓋在 Linux） 載體 Python＋C++＋Fennel 進度 step 1–5 全綠 · 3 語言互通 2026-07-19
一句話 外部用 ls / cat / echo 就能 walk、讀、寫「另一個正在執行的 process」的活狀態——因為它的執行態環境被暴露成路徑樹，且說的是真 9P，故可被 Linux 核心直接掛載、跨機器通聯。

01為什麼這樣做

推導出的「歸一於路徑」，其實就是 Plan 9 已做完的事——貝爾實驗室做 Unix 那批人，把「一切皆檔案」貫徹到 Unix 沒膽貫徹處。決定 方案 A：偷它的模型與 9P 協議，蓋在 Linux 上（FUSE/9P），保留現有生態；不 boot Plan 9 本體（留作以後逃生門）。設計脈絡在筆記鏈：

世界（OS-as-game）篇兩層底座重解篇實作路線規劃篇

02核心概念

歸一於路徑

不只靜態檔案；連正在執行的東西、呼叫方式都收斂成路徑可尋。

9P ＝通用協議

規範層從「你得是某語言」變「你得會講 9P」——天生語言中立。

ctl / data / status

每個元素是目錄，含三約定檔：data 讀寫、ctl 寫命令（取代 ioctl）、status 讀狀態。

兩種索引皆落成路徑

list→數字子路徑 /0 /1（0-based）；dict→字串鍵 /name /hp。

執行態非快照

讀到的是此刻狀態；發布端在 tick，同路徑不同時刻不同值；echo 改的是活物件本身。

/proc 是祖師爺

「把執行中 process 掛成路徑」是 /proc 既有範式，unipath 是它的最小自造版。

03·5多語言 port：語言中立實證

規範層宗旨＝「你得會講 9P，跟語言無關」。已用三語言各寫一版 9P server，並用同一個 Python 9P client 逐位元組對話驗互通：

Python ✅

參考實作（up_ninep*）。

C++ ✅

raw socket、g++、無 libfuse；逐位元組近乎全同。

Fennel ✅

05怎麼跑

指令都在 sub_projs/unipath/ 下；.venv 已建、裝好 fusepy。

# step 2 自掛（最簡）
.venv/bin/python unipath_live.py mnt
cat mnt/0/data ; cat mnt/0/data      # 兩次不同 → 執行態
echo 999 > mnt/2/hp/data ; cat mnt/2/data
fusermount -u mnt

# step 3 跨 process（三個終端）
.venv/bin/python unipath_pub.py /tmp/unipath.sock
.venv/bin/python unipath_mount.py /tmp/unipath.sock mnt
ls mnt ; echo 777 > mnt/2/hp/data ; fusermount -u mnt

# step 4 真 9P + 核心掛載（mount 需 root，自己跑）
.venv/bin/python unipath_9p.py serve 5640
.venv/bin/python unipath_9p.py selftest 5640   # 獨立 9P client 自測
sudo modprobe 9pnet_fd 9p
sudo mount -t 9p -o trans=tcp,port=5640,version=9p2000,uname=$USER 127.0.0.1 /tmp/umnt
ls /tmp/umnt ; cat /tmp/umnt/0/data ; sudo umount /tmp/umnt

# step 5 tick 示範
.venv/bin/python unipath_tick.py

# 多語言 port 跨語言互通（啟任一語言 server，用 Python client 打它）
( cd ports/cpp && make && ./unipath_9p_cpp serve 5661 )   # 或 ports/fennel: ./serve.sh 5662
.venv/bin/python unipath_9p.py selftest 5661            # → SELFTEST OK

06界線與下一步

誠實標註：step 3 是 9P 的「形狀」（JSON-over-socket），step 4 才是真線格式。真 9P 掛載需 root（mount 天生特權）；讀一定通，透過核心 mount 寫入涉及 9P uid 權限。暫忽略 CPU 週期（僅限慢世界）。Python 只是參考實作，規範層（講 9P）才是重點。
  • 使用者影響改走路徑樹寫入：把 tick 的旁路佇列改成吸收 echo > …/data 的樹寫入 → 「借樹＝影響」合一。
  • 巢狀 tick：上層目錄的一個 tick 驅動下層元素多次 tick（承筆記 §4.3）。
  • 規則換可定址腳本：tick 規則從硬編 kind 表換成路徑上可定址的腳本（NPC＝腳本）＝往階段二 os-as-game。
  • 真 9P 寫入 uid/權限、9P2000.L 擴充、發布端接真實 process 狀態（非示範 world）。
