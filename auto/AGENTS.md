## 系統概念

每個能力都是一個 **process**（可執行檔），遵守 `--metadata` 約定。
Hub 是一次性索引工具，不是 server。

## 你能做什麼

- 找已有 process：`hub --search-func "<keyword>"`
- 看單一 process 的 metadata：`./processes/<name>.py --metadata`
- 跑 process：`echo "input" | ./processes/<name>.py` 或 `./processes/<name>.py "input"`
- 新增 process：見 README §「AI 寫 process 的標準流程」六步

## Process 約定

- `--metadata` 旗標印一行 JSON，含必填 `name`、`description`，選填 `version`、`tags`、`input`、`output`、`examples`。
- 一般執行：CLI arg 或 stdin 二擇一接受輸入；成功寫 stdout exit 0；失敗寫 stderr non-zero exit。
- `examples` 內每個 `{input, output}` 都會被 `hub --validate` 自動執行比對，當作回歸測試。

## 進入專案後的建議流程

1. 跑 `hub --search-func "<關鍵字>"` 看有什麼可用；不確定時直接讀 `auto/FUNCTIONS.md`
2. 找到候選 process 後跑一次 `--metadata` 確認 io 約定
3. 編排：用 `processes/chain.py` 串連，或用 `processes/planner.py` 從自然語言任務產出計劃
4. 沒有合適的 process 時：跟使用者確認後，照六步流程新增一個
