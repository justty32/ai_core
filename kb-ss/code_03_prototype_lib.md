# code_03 — 原型 Library 集 (`try_implement/lib/`) 說明書

> **一句話講完**：這些是給工具用的「積木」（Module）。它們不是獨立的工具，而是讓開發者在寫工具時可以快速調用的功能庫。一樣**純 Python 內建**，沒裝任何 `pip` 套件。

---

## 1. 幫你打雜的（規範實作）

### `state_dirs.py` — 存檔路徑管家
- **幹嘛用的**：幫你決定資料要存哪。它會把檔案乖乖分進 `.config`（設定）、`.cache`（暫存）、`.state`（進度）、`.data`（成果）這四個資料夾。

### `recovery.py` — 當機救星
- **幹嘛用的**：如果程式跑一半斷掉，它會幫你記住最後的位置。下次啟動時，它能自動偵測到上次沒跑完，問你要「繼續（resume）」還是「重來（reset）」。

### `memoize.py` — 快取記憶
- **幹嘛用的**：如果輸入的東西一模一樣，就直接噴上次的結果，不要重算。主要用來省時間和省錢。

---

## 2. 基礎設施（底層功能）

### `server.py` — 伺服器框架
- **幹嘛用的**：幫你把一個小腳本變成一個「一直開著」的伺服器。支持標準的啟動、就緒、處理請求、關閉流程。
- **亮點**：支持 Unix Socket，讓多個工具可以連到同一個伺服器共用狀態（這對限流很重要）。

### `singleton.py` — 單例資源管理
- **幹嘛用的**：有些東西一次只能一個人用（例如打 LLM API）。它提供了「排隊（Queue）」和「用量計費（Rate Limit）」的功能。

### `trace.py` — 追蹤器
- **幹嘛用的**：當 A 叫 B，B 又叫 C 時，它會幫你印出一棵漂亮的「調用樹」，讓你知道到底是哪一步慢了或壞了。

### `call.py` — 萬能呼叫器
- **幹嘛用的**：不管你想呼叫同一個檔案裡的函式、啟動一個子程序、還是打一個 HTTP API，介面通通長一樣。

### `llm_call.py` — LLM 包裝器
- **幹嘛用的**：專門處理 LLM 調用。你可以隨便換後端（OpenAI, Anthropic, 或測試用的 Echo）。
- **特色**：支持「綁定（bind）」，你可以預先設定好 System Prompt，然後把它當成一個普通函式來用。

---

## 3. 組合術（最酷的部分）

### `compose.py` — 行為組合
- **幹嘛用的**：把函式像積木一樣疊起來。
- **招式**：
    - `pipe`：A 跑完給 B。
    - `retry_until_valid`：如果 LLM 胡言亂語，就叫它重講，直到講對為止。
    - `vote`：同一個問題問五次，取出現次數最多的答案（對付隨機性）。

### `interact.py` — 來回對話
- **幹嘛用的**：處理「一來一回」的邏輯。
- **招式**：
    - `actor_critic`：一個寫、一個批，批完再改，直到滿意。
    - `debate`：讓多個 AI 辯論，最後找個裁判（Judge）下結論。

### `compose_meta.py` — 自動推導 Metadata
- **幹嘛用的**：如果你把 A 和 B 串起來，這個模組能幫你自動算出來這條「新水管」的九軸 Metadata 是什麼（例如：只要其中一個是有狀態的，整條水管就是有狀態的）。
