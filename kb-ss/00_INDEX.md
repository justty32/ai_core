## 3. 一句話講完 ai_core

> ai_core 就是**要把 LLM 關進籠子裡**。我們只讓它做「翻譯意圖」和「模糊前沿處理」。預設是「不准進來」，要進來得憑證。然後靠「飛輪」把原本模糊的東西慢慢固化成穩定的 Code。

**主要目標**：寫 Code 助手（強的模型生結構化資產，弱的模型填空）。
**硬道理**：KISS、輕量、不重複造輪子、**零依賴**（只用 Python 3.11+ 內建庫，連 `pip install` 都不用）。

## 6. 碎碎念（發現的衝突）

整理時發現幾個點要注意：
1. **測試數字有三套**：核心測試 65 個、整體測試 84 個、原型煙霧測試 161 個。都對，只是範圍不同。
2. **八軸 vs 九軸**：舊文寫八軸，但**現在是九軸**（多了 `nondeterministic` 處理 LLM 的隨機性）。
3. **舊版 vs 新版**：舊版（④、⑤）跟現在的想法很多是相反的（以前愛用外部套件、愛常駐 Server），**現在堅持零依賴**。
