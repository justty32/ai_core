# roadmap.md

目標是可靠、速度、跨多面向、省錢、省資源(運算&空間)

為了可靠，所以要將對AI的呼叫包裝成函數，用盡可能少的input/output，與頻繁的呼叫次數去限制。
  再往上，那就是要逐步剃除對AI的依賴。
為了速度，所以要規範函數的自我描述，如此可以用時間限制當參數，去選擇更適合的函數。
  再往上，那就是逐步剃除、替換、優化較慢的函數。
  省資源也是首此影響。時間資源也是資源。
為了跨多面向，所以僅提供規範與標準實作。並且遵循Unix哲學。
  並且為了不重造輪子，所以會盡量去符合各種現有的AI運用，如MCP, skill, Dnspy
為了省錢，所以將AI的接口包裝起來成函數，並用函數的自我描述，描述其特性，盡可能最大化利用。
  所以訂下了初期使用較強AI，後期逐步替換成較笨AI的方向。
  剔除對AI的依賴也是受此影響。
-> 函數自我描述
-> 標準規範、實作、現有工具接口
-> 單函數、多函數、自我優化工作流

未來，想要建立一個類似於skill market那樣的工作流中心，搭上區塊鏈。
大家可以在此貢獻工作流，包括提出、應用修改等等，然後用區塊鏈紀錄貢獻。
然後靠捐獻給大家發錢。但貢獻除了考慮程式碼比例外，也要考慮使用者比例。

> 建立於 2026-05-24，源自一次「檢討初心」的討論；同日第二輪討論把重心從「隔開貴/便宜智能」搬到「**LLM 退縮到兩介面 + 拒絕為預設的憑證准入**」，已整合於下。
