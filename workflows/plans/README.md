**[spec](../spec/) 的實際實作規劃落點**：把已定案的設計規範，轉成排好開工次序與驗收標準的施工計畫。位於規劃管線的 **spec 定案 → plan → build** 之間。

## 這裡放什麼

- 某個 spec 要落地時的**施工單**：拆解步驟、依賴次序、驗收判據、要碰哪些檔。
- 與 [roadmap.md](../roadmap.md) 的分工：roadmap §6 給**戰略層的 v0 最小垂直切片**（該不該做、先做哪個）；本夾給**具體到可開工的工程規劃**（怎麼做、按什麼順序、怎麼驗收）。
