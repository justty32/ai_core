## 一、使用者原話盤點（三條路）

### 路徑一：把作業系統當程式語言
- **每個檔案＝一個函數**，在此基礎上做出 metadata 這東西。
- 要做的工程：把 Linux 的各種 IPC —— `argv/argc`、shared memory、`stdin/stdout/stderr`、pipeline、signal、socket、以 argv/argc 傳入的特定檔案 —— **統合成可供各種程式語言使用的 library**。
- 再往下：把 Linux 的一些特性（檔案位置處理／系統呼叫／行程的使用）也弄成**統一且方便存取**的形式。
- 理想載體＝**Plan 9**；但現在狀況恐怕沒辦法，只能退而求其次——**追求 Plan 9 哲學**。

### 路徑二：基於現有 cllm 與其開發環境
- 把 **LLM 呼叫當作函數**使用。
- 終極目標：**在 Lisp 裡，讓 LLM 參與 metaprogramming**。
- 與路徑一**不衝突**。
- 主要麻煩：把 **LLM 各方 API 整合成統一的呼叫方式**比較麻煩。
- **Lisp 本身就是一種作業系統**，只是要處理的東西也差不多。

### 路徑三：用 Lisp 去蒸餾 LLM（galtxt 那條路）
- 目標：**不使用神經網路與 GPU，就能達到 LLM 的效果**。
- **再往前一步**〔使用者追加・2026-07-18〕：跳出神經網路框架後，可以嘗試再往前，**去做出純 Lisp 的強化學習**。
- 與前兩個**不衝突**。

〔決定〕使用者明示：**三條路互不衝突**。

## 三、Claude 的結構性綜合〔提案〕：三條路＝同一個「語意 tracing JIT」的三個高度

接 §十三。三條路不是三個互斥的設計選項，而是**同一系統的三個正交層**——這正是它們「不衝突」的**結構原因，不是巧合**：

- **路徑一 ＝ 共用呼叫慣例／ABI 基底（鋪軌）**：把 OS 的 IPC 統一，讓每個 element——不管底層是 LLM 還是確定性 code——都用**同一種方式被呼叫**。＝「統一接口」那支柱在 OS 層的具體落地。
- **路徑二 ＝ interpreter／語意基準 tier（燒爐）**：把 LLM 做成可呼叫、可在 Lisp 裡組合的 **semantic oracle**（§13.3 的「語意基準面」）。
- **路徑三 ＝ compiler／tier-up（鑄刀）**：把 LLM 蒸餾成**非神經、確定性的 Lisp 碼**＝實際的固化。對映 §13.1 的 distill／codegen 階。

> 一句話：**路徑二燒爐、路徑三鑄刀、路徑一鋪它們共用的軌。** 三者在 JIT 架構裡各就其位，所以能並行不打架。

## 四、三個要當面講的判斷〔提案〕（風險／優先序，非替使用者決定）

1. **「不衝突」≠「同優先」。** roadmap 的「目標問題＝停止鍵」正是為了防止**在有消費者之前，把基底無限打磨**。路徑一（完整 IPC library＋Plan 9 野心）**最誘人也最危險**——它可以永遠做下去而始終沒有消費者，正是停止鍵要擋的那類 yak-shave（呼應 §十「metadata 稅」與適用邊界剎車）。

2. **路徑二的「整合各家 API 很麻煩」——這塊 cllm 其實已經做掉一大半。** `libcllm.so` 唯一入口 `llm_ask` 已統一吃 prompt/schema/tools/media/modalities/stream，並有九語言綁定＋端到端 smoke。剩下的主要是「**把 Lisp metaprogramming 那層接進來**」（`comfy/` 已備 Common Lisp 地基），不是從零整合 API。→ 路徑二是三條路裡**最接近能跑、風險最低**的，也正對 §12.4「前期以更好地使用 LLM 為主」。

3. **路徑三是整個賭注的本體，也是最高風險的 moonshot。** 「飛輪吃掉 LLM」的終局（§11.3 熵減／LLM 依賴單調遞減）就靠它兌現；但「不用 GPU／神經網路達到 LLM 效果」是**極強的主張**。它真正的價值是**驗證路徑二的飛輪會收斂**——沒有它，路徑二只是「一個更好用的 LLM 前端」，稱不上本專案獨有的立足點。

→ **一個可能的骨架〔提案・待使用者裁〕**：
- **路徑二當脊椎**：有在跑的消費者（cllm），先把爐子燒穩、把「LLM as function in Lisp」跑通。
- **路徑一按需拉入**：只在路徑二／三**實際喊出某個 IPC 需求**時才做那一塊，**不投機預建**，讓停止鍵守住邊界。
- **路徑三當北極星驗證**：moonshot，證明飛輪真的會終止；早期以「先蒸出一把能用的小刀」為里程碑，不追「絕對歸零」。
